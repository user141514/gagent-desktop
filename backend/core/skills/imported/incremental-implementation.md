## Intent

Use this skill when the user asks for a minimal implementation step, low-risk progress, or explicitly rejects a broad refactor.

## Minimal SOP

1. Keep scope narrow and end-to-end.
2. Make one concrete change that can be verified immediately.
3. Preserve existing behavior outside the target path.
4. Add only the supporting code required for the current step.
5. Stop after the smallest complete increment and report follow-up options.
